Introducci√≥n

En un problema de localizaci√≥n se busca encontrar la ubicaci√≥n √≥ptima de un servicio, o un conjunto de ellos, de forma que la calidad que dicho servicio presta a un conjunto de puntos de demanda sea, seg√∫n cierta medida, √≥ptimo. Algunos ejemplos de problemas de localizaci√≥n son:

  • Encontrar la ubicaci√≥n √≥ptima del almacen central de una red de distribuci√≥n de mercanc√≠as para que se minimice el costo total de transporte
  • Encontrar la ubicaci√≥n √≥ptima de una ambulancia que debe atender a los pacientes de cierta regi√≥n para que se minimice el tiempo en atender al paciente m√°s lejano

Son numerosos los contextos en los que se plantean problemas de localizaci√≥n, debido a ello la teor√≠a de localizaci√≥n ha sido objeto de gran atenci√≥n en los √∫ltimos a√±os, pudiendo decirse que es un tema de gran actualidad y vigencia. A ello contribuye la aparici√≥n de facetas del problema hasta ahora no estudiadas. Por ejemplo, junto a los ya cl√°sicos criterios de minimizaci√≥n de costos, aparecen nuevos criterios: ambientales, sociales, calidad de vida, etc. Estas nuevas vertientes del problema hacen que sea un campo abierto de estudio.

El paquete que se presenta est√° dedicado a la resoluci√≥n del problema de localizar un √∫nico punto en el plano usando como objetivo la minimizaci√≥n de la suma de las distancias ponderadas a los puntos de demanda. Nuevas versiones del paquete incluir√°n nuevos modelos de localizaci√≥n.

La clase de objetos loca.p

En un problema de localizaci√≥n plana el conjunto de puntos de demanda viene dado por las coordenadas de dichos puntos. Opcionalmente, se puede asignar a dichos puntos una ponderaci√≥n, que da mayor importancia a unos puntos que a otros, dado que el objetivo que se considera es minimizar la suma ponderada de las distancias entre el punto de servicio y dicho conjunto demandante. Por ejemplo, si se busca la localizaci√≥n de un hospital comarcal, los puntos de demanda pueden ser las localidades a las que el hospital debe atender y las ponderaciones la poblaci√≥n de cada localidad.

Para la resoluci√≥n de estos problemas se ha definido una clase de objetos designada loca.p, de forma que un objeto loca.p almacena las coordenadas de los puntos de demanda y las ponderaciones de cada uno de los puntos. Cada objeto loca.p tiene tres slots, x e y que almacenan las coordenadas y w que almacena las ponderaciones. Cuando las ponderaciones no se den de forma expl√≠cita, se considerar√° que todos los puntos de demanda tienen igual importancia.

En el resto de esta secci√≥n se expondr√° la forma de hacer las operaciones b√°sicas con objetos loca.p.

Creaci√≥n de objetos de clase loca.p

Consideremos un problema de localizaci√≥n en el que el conjunto de puntos de demanda es \((0,0)\), \((4,0)\) y \((2,2)\). Para crear un objeto loca.p que represente a dicho conjunto, se puede hacer llamando a la funci√≥n constructora usando como argumentos el vector con las coordenadas \(x\) y el vector con las coordenadas \(y\) del conjunto de puntos:

#>

#> [1] 1 1 1
#> 
#> Slot "label":
#> [1] ""

o alternativamente:

El constructor tiene dos argumentos opcionales m√°s, el tercero w se usa para especificar un vector de pesos y el cuarto para especificar una etiqueta que se usar√° para identificar el objeto. Si, usando el mismo conjunto de puntos, se quiere asignar los pesos 1, 1, 3, a dichos puntos y la etiqueta “Problema 1”, se usa:

#> [1] 1 1 3
#> 
#> Slot "label":

Rec√≠procamente, un objeto loca.p se puede convertir en un objeto data.frame mediante:

Distancia media ponderada

Dado un objeto loca.p se puede evaluar la distancia ponderada desde un punto dado. As√≠ mismo, se puede evaluar el gradiente de dicha funci√≥n y se puede resolver el problema de minimizar dicho objetivo.

Evaluaci√≥n

La funci√≥n distancia media ponderada se denomina en el paquete distsum, dado un punto, por ejemplo: \((3, 1)\) se puede evaluar la distancia media ponderada a un objeto loca.p:

pt3 <- loca.p(x = c(0, 4, 2), y = c(0, 0, 2), label = "Tres puntos")
distsum(o = pt3, x = 3, y = 1)
#> [1] 5.990705

Tambi√©n se puede calcular el gradiente de distsum llamado distsumgra:

distsumgra(o = pt3, x = 3, y = 1)
#> [1] 0.9486833 0.3162278

Resoluci√≥n

Para encontrar la soluci√≥n √≥ptima al problema de localizaci√≥n anterior se usa la funci√≥n distsummin:

s <- distsummin(pt3)
s
#> [1] 2.00000 1.15332

Evaluando la funci√≥n y el gradiente en el punto obtenido

distsum(o = pt3, x = s[1], y = s[2])
#> [1] 5.464102
distsumgra(o = pt3, x = s[1], y = s[2])
#> [1]  3.110246e-07 -8.970172e-04

Como se puede comprobar por el valor del gradiente, la soluci√≥n encontrada es un √≥ptimo local y al ser la funci√≥n objetivo convexa un √≥ptimo global.

Estas tres funciones admiten un argumento opcional lp, si se omite este argumento, se utiliza la norma eucl√≠dea, es decir, la norma \(l_2\), si se especifica un valor para lp se utilizar√° la norma \(l_p\) para dicho valor de \(p\).

Obs√©rvese que si se especifica lp = 2 se utiliza el algoritmo gen√©rico para la norma l_p con \(p\) igual a 2. La utilizaci√≥n del algoritmo gen√©rico requiere un mayor esfuerzo computacional para la resoluci√≥n del problema, por lo que no es recomendable especificar dicho argumento para usar la norma eucl√≠dea.

Dibujando

Tanto los objetos loca.p como la funci√≥n objetivo pueden representarse en un gr√°fico. Para la funci√≥n objetivo se proporciona una representaci√≥n basada en curvas de nivel y otra en un gr√°fico 3D.

Dibujar un objeto loca.p

La gr√°fica de un objeto loca.p consiste en representar en el plano el diagrama de dispersi√≥n del conjunto de puntos de demanda usando la funci√≥n plot:

plot(pt3)

Gr√°fico de curvas de nivel

El gr√°fico de curvas de nivel se realiza con la funci√≥n contour:

contour(pt3)

En el gr√°fico se puede observar c√≥mo la funci√≥n alcanza el m√≠nimo en el punto calculado anterioremente. Ampliando:

contour(pt3, xlim = c(1.9, 2.1), ylim = c(1, 1.2), levels = c(5.465, 5.47, 5.475))

Las funciones plot y contour admiten un argumento opcional img que permite especificar un gr√°fico raster que se usar√° como fondo del gr√°fico.

Gr√°fico en 3D

An√°logamente se puede realizar una representaci√≥n en tres dimensiones usando la funci√≥n persp:

persp(pt3, col = "lightblue", theta = 45, ltheta = 120, shade = 0.75, ticktype = "detailed")

Las tres funciones de representaci√≥n pasar√°n los restantes argumentos opcionales a la funci√≥n gen√©rica plot.

Ejemplo de localizaci√≥n en Andaluc√≠a

Se cargan los datos de las capitales andaluzas y se convierte en un objeto de clase loca.p:

o <- loca.p(x=andalusia$x[1:8], y=andalusia$y[1:8])

Se calculan los valores l√≠mite para el gr√°fico:

xmin <- min(andalusia$x)
ymin <- min(andalusia$y)
xmax <- max(andalusia$x)
ymax <- max(andalusia$y)

Se carga el mapa de Andaluc√≠a y se representan los puntos con el mapa de fondo

file = system.file('img', 'andalusian_provinces.png', package='orloca')
img = readPNG(file)
plot(o, img=img, main='Andaluc√≠a', xleft=xmin, ybottom=ymin, xright=xmax, ytop=ymax)

El gr√°fico de curvas de nivel es:

contour(o, img=img, main='Andaluc√≠a', xleft=xmin, ybottom=ymin, xright=xmax, ytop=ymax)

La soluci√≥n √≥ptima del problema de localizaci√≥n con las 8 capitales, ocho primeras filas, se obtiene:

andalusia.loca.p <- loca.p(andalusia$x[1:8], andalusia$y[1:8])
sol <- distsummin(andalusia.loca.p)
sol
#> [1] -4.610679 37.248691

La soluci√≥n √≥ptima que proporciona el algoritmo est√° localizada a unos 35 Km al norte de Antequera. Recu√©rdese que usualmente se considera a Antequera como el centro geogr√°fico de Andaluc√≠a. El gr√°fico presenta la soluci√≥n como un punto de color rojo:

points(sol[1], sol[2], type='p', col='red')

Por simplicidad en el ejemplo, no se ha tenido en cuenta la curvatura terrestre.
